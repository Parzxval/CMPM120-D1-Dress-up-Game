GitHub Copilot: 

The issue is that `this.graphics.fillRect()` returns a `Graphics` object, not a Game Object with interactive capabilities. The `setInteractive()` method doesn't work on Graphics objects the same way it does on Sprites or Shapes.

## Fix

Use `this.add.rectangle()` instead, which creates a Shape object that supports interactivity:

```javascript
create() 
{
    this.load.path = 'assets/';
    this.load.image('firstBackdrop', 'firstBackdrop.png');

    this.firstBackdrop = this.add.image(675, 350, 'firstBackdrop');
    this.firstBackdrop.setScale(1.4);

    // Use add.rectangle instead of graphics.fillRect
    this.newButton = this.add.rectangle(190, 367.5, 320, 75, 0x104574ff);
    
    this.newButton.textObject = this.add.text(50, 350, 'Present your game...',
        {font: "30px Arial", color: "#dddddd"}
    );
    
    this.newButton.setInteractive({useHandCursor: true});
    this.newButton.on('pointerdown', () => this.scene.start('Scene2'));
}
```

**Key changes:**
- `this.add.rectangle(x, y, width, height, color)` creates a Shape with built-in interactivity
- Adjusted x/y to center the rectangle (30 + 320/2 = 190, 330 + 75/2 = 367.5)
- Removed the `graphics.fillGradientStyle` line since `add.rectangle` doesn't support gradient fills directly—you'd need a different approach for that

## How to Remove Old Hairstyle

The issue is that `forwardClicked` creates a new image every time but never removes the previous one. You need to store a reference to the current image and destroy it before adding a new one.

### Fix

```javascript
create()
{
    // ...existing code...
    
    this.hairstyles = ['hair1', 'hair2', 'hair3'];
    this.currHairIndex = 0;
    this.currentHair = null; // Track current hair image

    this.forwardButton.on('pointerdown', () => {
        console.log('forward button clicked');
        
        // Remove old hairstyle first
        if (this.currentHair) {
            this.currentHair.destroy();
        }
        
        this.forwardClicked(this.hairstyles[this.currHairIndex]);

        if (this.currHairIndex < this.hairstyles.length - 1) 
        {
            this.currHairIndex++;
        } else 
        {
            this.currHairIndex = 0;
        }
    });
}

forwardClicked(img) 
{
    this.currentHair = this.add.image(-650, -350, img);
    this.currentHair.setScale(0.19);

    this.tweens.add({
        targets: this.currentHair,
        x: 650,
        y: 350,
        duration: 200,
        ease: 'Linear',
    });
}
```

### Key Changes

1. **Added `this.currentHair = null`** - Initialize a variable to track the current hairstyle image
2. **Added cleanup in click handler** - Before adding a new hairstyle, check if `this.currentHair` exists and call `.destroy()` to remove it
3. **Store reference** - Assign the new image to `this.currentHair` so it can be removed next time

GitHub Copilot: 

## Explanation

This line of code attempts to toggle the switch button's texture between two different button images when clicked. However, there are two issues with this code:

### 1. Wrong Method Name

The `.replace()` method doesn't exist in Phaser 3 for changing an image's texture. You should use `.setTexture()` instead.

### 2. Logic Issue

The ternary operator checks `this.assetType % 2 === 0`, but since `assetType` is incremented after every click, the button will always show the opposite of what you might expect. When `assetType` is 0 (even), it shows `'switchHairButton'`, then increments to 1. On the next click, `assetType` is 1 (odd), so it shows `'switchClothesButton'`, then increments to 2. This means the button always shows the *next* asset type, not the current one.

### Fix

```javascript
this.switchButton.on('pointerdown', () => {
    console.log('switch button clicked');
    
    // Use setTexture instead of replace
    // Show current asset type (before incrementing)
    this.switchButton.setTexture(this.assetType % 2 === 0 ? 'switchHairButton' : 'switchClothesButton');
    
    this.assetType++;
});
```

This way, when the button is clicked, it displays the image corresponding to the *current* asset type, then switches to the other type for the next click.
